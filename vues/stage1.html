Mon stage de 1re année chez Kpi Technologie.


L'entreprise

Kpi Technologie est une entreprise qui est spécialisée dans la réalisation de systèmes d'informations industriels. Kpi Technologie accompagne ses clients dans la mise en place de projets structurants et novateurs. L'entreprise intervient également pendant les phases d'étude, gestion de projet, développement en support.

Les collaborateurs

Michel Trompier : Créateur et directeur de Kpi Technologie.
Remi Hommet : Développeur full stack qui m'encadrera tout au long du stage.
Baptiste Bodin : Stagiaire avec qui je vais développer l'application.

Le contexte

Mr Trompier souhaite créer une application web qui permettrait aux entreprises de gérer leurs contrats.

Technologies utilisées


Microsoft Sql Server

C’est un système de gestion de base de données en langage SQL incorporant entre autres un SGBDR développé et commercialisé par la société Microsoft.

Microsoft Visual Studio

C’est une suite de logiciels de développement pour Windows et Mac OS conçue par Microsoft. La dernière version s'appelle Visual studio 2022.

ASP.NET Core

C’est un Framework Web gratuit et open source, développé par Microsoft et la communauté. Il est plus performant qu'ASP.NET. C'est un Framework modulaire qui fonctionne à la fois avec le Framework .NET, sous Windows et .NET Core en multiplate-forme. Il permet le développement d’applications web.

Bootstrap

C’ est une collection d'outils utiles à la création du design de sites et d'applications web. C'est un ensemble qui contient des codes HTML et CSS, des formulaires, boutons, outils de navigation et autres éléments interactifs, ainsi que des extensions JavaScript en option.

Ajax

Ajax est une méthode utilisant différentes technologies ajoutées aux navigateurs web entre 1995 et 2005, et dont la particularité est de permettre d'effectuer des requêtes au serveur web et, en conséquence, de modifier partiellement la page web affichée sur le poste client sans avoir à afficher une nouvelle page complète

Développement de l'application


Première semaine

Durant les premiers jours, j'ai découvert l'infrastructure informatique de l'entreprise. Celle-ci comporte: un serveur avec deux processeurs accompagné de 8 disques en raid-5 d’une capacité de stockage d’environ 1,2 To, le serveur est également composé d’environ 100 Go de RAM. La principale utilité du serveur est de stocker les machines virtuelles. En effet, chez KPI Technologies le développement se fait sous machines virtuelles. Pour se connecter à elles, on utilise des clients légers(voir image à droite).

Cahier des charges

Nous avons parlé du cahier des charges. Étant donné que l’application sera un produit proposé par l'entreprise, le cahier des charges est relativement souple car il ne nous est pas imposé. L’application est un outil visant à gérer des contrats pour les grosses entreprises, avec une vue d’ensemble sur ceux-ci et une description plus détaillée si on le souhaite. Une notification quand un contrat arrive à expiration et chaque utilisateur de l’application aura un rôle bien défini.

Nous avons commencé à programmer la page de connexion d’un utilisateur sans les rôles afin que la liaison base de données-application soit fonctionnelle. Nous utilisons également Bootstrap pour tout ce qui est Développement Web Frontal.

On a créé un petit menu de navigation en CsHTML et js avec bootstrap en utilisant des vues appelées “modal”.

Deuxième semaine

J’ai fait la connaissance de Baptiste, un autre stagiaire (nous sommes 2 stagiaires en tout). durant la matinée je lui ai présenté ce que nous avons fait pendant la première semaine de stage, le projet, et l’infrastructure informatique. Afin de gagner du temps, nous nous sommes réparti les tâches, lui fera la partie “backend” (en grande partie du c# et gestion de la base de données) et moi le “frontend” ( beaucoup d’HTML, Css et bootstrap ainsi que du JavaScript) Nous avons commencé à développer ensemble la fin du menu de navigation et supprimer la page d’accueil. La page d’accueil sera désormais une page de connexion.

Nous avons créé une page dans le menu, accessible uniquement avec le rôle administrateur, qui permet de voir l’ensemble des utilisateurs de l'application et leurs rôles. Nous y avont ajouté les fonctionnalitées modifier et supprimer sur la droite.

On a également crée un formulaire qui permet d’ajouter un utilisateur à la base de données avec son rôle.

Quatrième semaine

Cette semaine nous commençons une nouvelle page un peu dans le même esprit que celle qui montre la liste des utilisateurs. Cette page appelée "Consultation des contrats" montrera l’ensemble des contrats disponibles sur l’application en fonction du rôle de celui qui veut les consulter. Chaque ligne du tableau correspond à 1 contrat.

Dans un but pratique nous avons fait le choix dans un premier temps de masquer les données secondaires du contrat, si un utilisateur veut avoir accès à ces données, il devra alors cliquer sur la flèche bleue propre à chaque contrat qui fera apparaître toutes les données secondaires de celui-ci.

Cinquième semaine

Nous avons une page gestion de contrat qui s'affiche lorsque l'on clique sur un contrat sur la page consultation. Cette page va permettre de gérer le contrat sélectionné

Nous avons crée plusieurs formulaires afin de pouvoir ajouter des données dans les événements du contrat, les contacts du contrat, les documents du contrat et aussi les utilisateurs. On a également terminé la suppression pour chaque catégorie sauf utilisateur (cas particulier).

Sixième semaine

On crée des formulaires de modification pour chaque catégorie sauf documents (cas particulier).

Septième semaine

On passe ensuite au formulaire de création de contrats, cette page fera également la modification des contrats existants.
